📋 TFL QC Checklist

For Junior Biostatisticians (Clinical Trials)

Objective (one sentence):
Ensure Tables, Figures, and Listings (TFLs) are statistically correct, internally consistent, clearly documented, and inspection-ready.

🟦 0. Pre-QC Setup (Do this first otherwise QC is meaningless)

☐ I understand the purpose of this TFL:

  • ☐ Primary endpoint
  • ☐ Secondary endpoint
  • ☐ Exploratory analysis

☐ I know the QC level required:

  • ☐ Independent programming QC
  • ☐ Targeted QC
  • ☐ Visual QC

☐ I have the following current documents:

  • ☐ Latest SAP (version: ______)
  • ☐ TFL Shell / Specification
  • ☐ ADaM dataset description

🟦 1. Input Data QC (Most critical and most overlooked)

1.1 Analysis Population

☐ Correct analysis population used:

  • ☐ ITT / FAS
  • ☐ Safety (SAF)
  • ☐ Per-Protocol (PP)

☐ Population flags align with SAP definitions
☐ Total N matches other TFLs / CSR (if available)

1.2 Analysis Dataset

☐ Correct ADaM dataset and version used
☐ Data cut-off date ≤ Database Lock Date
☐ Key variables present and valid:

  • ☐ Treatment group
  • ☐ Baseline value
  • ☐ Visit / time point
  • ☐ Endpoint variable

🟦 2. Statistical Method QC (Check against SAP, not code)

☐ Analysis method matches SAP:

  • ☐ Model type (e.g., ANCOVA, MMRM, Cox, Logistic)
  • ☐ Covariates correctly included
  • ☐ Stratification factors applied if specified

☐ Missing data handling matches SAP:

  • ☐ MMRM
  • ☐ LOCF
  • ☐ Multiple Imputation
  • ☐ Non-Responder Imputation

☐ Analysis visit / window is correct

📌 Reminder:

If you do not fully understand the method, stop and ask a senior statistician.

🟦 3. Numerical QC (Check every number)

3.1 N and Denominators

☐ N values are correct for the analysis population
☐ Percentage denominators are correct
☐ 0% and 100% are displayed correctly

3.2 Descriptive Statistics

☐ Mean / SD / Median / Min / Max are reasonable
☐ Baseline and Change from Baseline are correctly defined
☐ Units are correct and consistent

3.3 Inferential Statistics

☐ Effect estimates align with confidence intervals
☐ CI level is correct (e.g., 95%)
☐ p-values:

  • ☐ Not rounded to 0.000
  • ☐ Display precision follows TFL specification

🟦 4. Cross-TFL Consistency QC (High inspection risk area)

☐ Same endpoint is consistent across:

  • ☐ Table vs Figure
  • ☐ Table vs Listing

☐ Same population is consistent across:

  • ☐ Baseline tables
  • ☐ Efficacy tables

☐ CSR text (if available) matches TFL results

🟦 5. Format and Text QC (Statisticians own the words too)

☐ Title clearly states:

  • ☐ Endpoint
  • ☐ Population
  • ☐ Analysis method

☐ Footnotes include:

  • ☐ Missing data handling
  • ☐ Model and covariate description

☐ All abbreviations are defined
☐ Decimal places follow TFL specification
☐ Missing values are displayed consistently

🟦 6. QC Documentation (If its not documented, it didnt happen)

☐ QC performed by: __________________
☐ QC date: __________________

☐ QC method:

  • ☐ Independent programming
  • ☐ Manual calculation
  • ☐ Visual review

☐ All issues identified have been resolved
☐ All discrepancies are documented and closed

🟦 7. Final Self-Check (Before delivery)

☐ I can explain where every number comes from
☐ The results make scientific and statistical sense
☐ I am confident answering QA or inspection questions

If any answer is NO, do not deliver the TFL.

🎯 One sentence for juniors (please share exactly as written)

QC is not about finding mistakes it is about preventing future inspection findings.
